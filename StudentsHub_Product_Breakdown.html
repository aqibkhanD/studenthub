System Architecture Overview
StudentsHub is a 5-layer system. Students and staff interact via a browser or mobile app (Vue/React). Every request flows through the Laravel API, optionally through the Python FastAPI AI layer, and persists in PostgreSQL. Redis handles queues and caching. All services are containerised with Docker Compose.

7 Intent Types — How the AI Classifies Every Message

Every student message is classified into one of these intents before routing to the correct agent or action path.

Information Request — "What are the graduation requirements?" Transaction — "I need to apply for a bonafide certificate" Status Check — "Where is my transcript request?" Escalation — "My request has been pending for 2 weeks" Sensitive — "I want to report a harassment incident" Opportunity — "What scholarships am I eligible for?" Governance — "I want to propose a new student club"

Deployment Assessment — Can This Be Deployed?
Yes — fully deployable. The Option C stack (Laravel + FastAPI + PostgreSQL + pgvector + Vue/React + Redis + Docker Compose) runs on any standard Linux VPS or cloud provider. Minimum viable deployment can be live in under 4 hours with Docker Compose. All components are open-source or have free tiers available.
📦 Stack Components
  • Laravel 11 (PHP 8.3) — Ready
  • Python 3.11 + FastAPI — Ready
  • PostgreSQL 16 + pgvector — Ready
  • Redis 7 — Ready
  • Vue.js / Next.js — Ready
  • Nginx (reverse proxy) — Ready
  • Docker Compose v2 — Ready
  • SSL Wireless SMS API — Need account
  • OpenAI / Anthropic API — Need API key
💻 Minimum Server Requirements
  • CPU: 4 vCPUs (2 minimum for dev)
  • RAM: 8 GB (4 GB minimum for dev)
  • Storage: 80 GB SSD (uploads + DB)
  • OS: Ubuntu 22.04 LTS
  • Network: 100 Mbps, static IP
  • Bandwidth: 2 TB/month (50k users)
  • Database: PostgreSQL must have pgvector extension enabled
  • Ports needed: 80, 443 (Nginx), 22 (SSH)
🚀 First Deployment Steps
  • 1. Provision VPS / server
  • 2. Install Docker + Docker Compose
  • 3. Clone repository, configure .env
  • 4. Run docker compose up -d
  • 5. Run Laravel migrations + seed
  • 6. Configure Nginx with SSL (Let's Encrypt)
  • 7. Point domain DNS to server IP
  • 8. Test SMS gateway connectivity
  • 9. Upload initial knowledge base documents
  • 10. Create super-admin account
⚠ Phase 1-3 Requirements (Beta)
  • No AI API key needed until Phase 4
  • No pgvector needed until Phase 4
  • Redis optional until Phase 3 (queue jobs)
  • SMS gateway needed from Phase 3
  • 4 GB RAM sufficient for Phase 1-3
  • Single VPS handles Phase 1-3 comfortably
  • Cost for beta: ~$20-25/month

Deployment verdict: The stack is battle-tested, fully open-source (except AI APIs), and deployable by one developer in a single afternoon using the Docker Compose file. The 90-day beta target (Phases 1-3) is achievable on a $25/month Hetzner VPS. Full production scale with AI requires upgrading to 8 GB RAM and adding pgvector — a one-line change in the Docker Compose config.
